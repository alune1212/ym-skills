# cn-city-wage-social-base

## 用途

这个 Skill 用于查询中国大陆单个城市当前有效的月最低工资和社保缴费基数上下限，并输出带摘录原文和摘录来源直链的 Markdown 报告。

它适合以下场景：

- 人事、薪酬、财务需要核验某个城市的最低工资
- 用户明确要求查询某个城市的社保基数或缴费基数上下限
- 用户要求“附原文摘录”“给直达官方链接”“最好给附件原文”
- 用户只给出区县、新区、开发区名称，但明确要查最低工资或社保基数

## 设计目标

- 只做一个城市，但把证据链做扎实
- 优先使用市人民政府官网，必要时才回退到市级主管部门官网
- 每个结论都要能被“摘录原文 + 摘录来源直链”支撑
- 遇到口径冲突或来源不完整时，宁可标记为“未确认”，也不猜测

## 默认口径

- 地域范围：仅中国大陆城市
- 查询范围：单城市
- 最低工资：只取月最低工资
- 社保基数：默认返回社保缴费基数下限和上限，不按险种拆分
- 时间口径：当前有效，不做历史版本对比
- 最终产物：Markdown 报告

## 明确的触发边界

这个 Skill 只在用户明确提到以下指标时触发：

- 最低工资
- 社保基数
- 缴费基数

以下近邻场景不应触发：

- 泛化薪酬分析或收入分析
- 劳动法咨询、法条解释、争议分析
- 省级或全国汇总
- 多城市比较
- 只要求找官网入口、政策网站入口

## 典型触发示例

- “查询苏州市月最低工资和社保基数，并给我官方出处。”
- “帮我查浦东新区的最低工资和社保缴费基数上下限，输出 Markdown 报告。”
- “我要做薪酬测算，先帮我核一下佛山的最低工资和社保基数，链接要直达原文。”

## 不应触发示例

- “帮我做杭州薪酬水平分析，看看研发岗平均工资。”
- “解释一下最低工资法条对加班费的影响。”
- “查广东省最低工资标准汇总。”
- “比较苏州和无锡的收入水平。”

## 输入与处理方式

- 直接城市名：按该城市检索
- 区县、新区、开发区：如果归属明确，自动归一到所属地级市或直辖市，并在报告中说明
- 省份、模糊地名、多个城市：先追问
- 香港、澳门、台湾及其他非中国大陆城市：直接说明不适用

## 输出结构

报告骨架见 [`assets/report-template.md`](assets/report-template.md)。

报告必须包含：

- 查询摘要表
- 城市识别说明
- 月最低工资
- 社保缴费基数上下限
- 摘录原文
- 摘录来源直链
- 公告页链接（可选）
- 查询说明与未确认项

## 来源策略

来源优先级见 [`references/source-priority.md`](references/source-priority.md)。

核心原则：

- 先找市人民政府官网正文或附件
- 找不到再回退到市级主管部门官网
- 不把第三方聚合站、论坛、短视频、招聘站、百科词条当作依据
- 不把“看起来像官方”的转载页面当作最终结论，必须追到原始文件或正式公告

## 证据对齐规则

- 摘录原文来自哪里，摘录来源直链就必须指向哪里。
- 如果真正数值在附件里，报告必须给附件直链；公告页链接最多作为补充，不得替代附件直链。
- 如果正文已经直接写明数值和适用范围，就引用正文并给正文 URL。
- 如果没有可直接摘录的原文，说明证据不足，标记为“未确认”。

## 为什么不使用非官方聚合站

最低工资和社保基数属于政策口径，年度更新和地方差异都很常见。聚合站常见问题包括：

- 未注明文件版本或生效时间
- 混用省级与市级口径
- 只给结论，不给原始文件链接
- 被转载后内容过时但页面仍可搜索到

这个 Skill 的目标不是“猜一个大致值”，而是产出可复核的政策报告，因此必须以官方站点为准。

## 市政府官网缺失时的处理

- 如果市政府官网只有摘要或转发说明，继续追到它引用的附件或原始文件
- 如果市政府官网没有直接信息，回退到市级人社局、医保局、税务等主管部门官网
- 如果仍然缺失、冲突或无法确认适用范围，在报告中明确写“未确认”，并说明已经检查过的来源类型
