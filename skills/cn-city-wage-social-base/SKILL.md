# cn-city-wage-social-base

## 目标

帮助用户围绕一个中国大陆城市完成政策核验，输出一份可复核的 Markdown 报告，覆盖：

- 当前有效的月最低工资
- 当前有效的社保缴费基数下限和上限
- 与结论逐项对齐的摘录原文
- 摘录来源直链，以及必要时补充的公告页链接
- 支撑结论的适用说明

## 触发条件

- 用户明确要求查询中国大陆单个城市的最低工资
- 用户明确要求查询中国大陆单个城市的社保基数或缴费基数上下限
- 用户给出区县、新区、开发区名称，并且明确要求查询最低工资、社保基数或缴费基数
- 用户明确要求“给原文摘录”“给直达官方链接”“给附件原文直链”，前提仍然是问题本身明确涉及最低工资或社保/缴费基数

## 不应触发的场景

- 泛化薪酬分析、薪资水平分析、招聘定价、收入对比
- 劳动法咨询、法条解释、合规意见或争议分析
- 省级汇总、全国汇总、趋势解读、历史沿革
- 多城市横向比较
- 只要求“找官网入口”“找政策网站”，但没有明确查询最低工资、社保基数或缴费基数

如果用户问题落在这些近邻场景，不要强行套用本 Skill。

## 先决判断

- 只支持单个中国大陆城市，不做多城市横向比较。
- 如果用户没有给城市、只给省份、给了多个城市，或地名无法稳定归一，先追问，再继续。
- 如果用户给的是香港、澳门、台湾或中国大陆以外城市，明确说明本 Skill 不适用。
- 如果用户给的是区县、新区、开发区且行政归属明确，先归一到所属地级市或直辖市，并在报告中写清：
  - 原始输入
  - 归一化结果
  - 为什么可以归一
- “最低工资”默认只查月最低工资，不展开小时最低工资，除非用户额外要求。
- “社保基数”默认指社保缴费基数下限和上限，不按险种拆分；如果官方文件只给单一口径或不同险种口径不一致，照实说明，不要自行补齐。
- 如果用户问的是省级范围、全国范围或多城市比较，不要输出拼接式答案；说明当前 Skill 只支持单城市，并要求用户缩小范围。

## 工作流

1. 从用户请求中提取城市候选、时间口径和输出要求。
2. 先判断城市是否合法：
   - 缺失、模糊、非中国大陆或多城市时先追问
   - 区县/新区/开发区能稳定归一时，记录归一关系后继续
3. 读取 [`references/source-priority.md`](references/source-priority.md)，按固定优先级检索官方来源。
4. 对每个待确认事项先找到“真正承载数值的目标页面”：
   - 如果政府公告页正文已经给出数值和适用范围，目标页面就是公告正文
   - 如果政府公告页只是转发摘要，真正数值在附件、PDF 或下游原文，目标页面就是附件或原文页
5. 检索“当前有效”的月最低工资来源：
   - 优先市人民政府官网正文或附件
   - 缺失时回退到市级人社局、医保局、税务等主管部门官网
6. 检索“当前有效”的社保缴费基数上下限来源：
   - 优先市人民政府官网正文或附件
   - 缺失时回退到市级主管部门官网
7. 对每个结论先建立证据链，再写结论。每条证据链至少包含：
   - 数值与单位
   - 文件标题或页面标题
   - 发文日期或公布日期
   - 生效时间或适用年度
   - 摘录原文
   - 摘录来源直链
   - 公告页链接（只有当摘录来自附件，且公告页有分发价值时才补充）
8. 对每个结论再次检查以下信息：
   - 确认摘录原文和摘录来源直链来自同一个目标页面
   - 不接受“摘录来自附件，但链接只给栏目页或摘要页”的写法
9. 如果同一事项出现多个官方口径：
   - 优先级高的来源优先
   - 同级来源时，优先采用生效时间更新、且表述更明确的文件
   - 仍无法判断时，不输出确定结论，转为“未确认”
10. 按 [`assets/report-template.md`](assets/report-template.md) 生成最终 Markdown 报告。

## 证据对齐规则

- 先确定摘录来源，再写报告；不要先写结论再回头找链接。
- 摘录原文必须能在最终给出的摘录来源直链里直接找到。
- 如果摘录来自 PDF 或附件，摘录来源直链就必须指向该 PDF 或附件本身。
- 如果摘录来自政府公告正文，就把公告正文 URL 作为摘录来源直链。
- 如果需要补充公告页链接，只把它作为可选补充字段；它不能替代摘录来源直链。
- 如果找不到可直接摘录的原文，或链接与摘录无法一一对应，把该项标记为“未确认”。

## 输出要求

- 输出格式固定为 Markdown。
- 报告至少包含以下章节：
  - 标题
  - 查询摘要表
  - 城市识别说明
  - 月最低工资
  - 社保缴费基数上下限
  - 摘录原文
  - 摘录来源直链
  - 公告页链接（可选）
  - 查询说明与未确认项
- 每个数值都必须绑定到一个官方来源；没有来源就不要给结论。
- 每个已确认结论都必须带上以下字段：
  - 文件标题或页面标题
  - 摘录原文
  - 摘录来源直链
- 摘录保持简短，优先摘录最能支撑数值、适用范围和生效时间的原文。
- 摘录原文与摘录来源直链不一致时，视为无效证据。
- 如果某项无法确认，明确写“未确认”，并说明原因。

## 追问规则

- 没有城市时：
  - “请提供要查询的中国大陆城市名称；如果你只有区县或新区名称，也可以直接告诉我。”
- 只给省份时：
  - “请补充具体中国大陆城市名称；当前口径按单个城市查询，省级范围无法直接替代。”
- 非中国大陆城市时：
  - “这个 Skill 只支持中国大陆城市。请改为提供一个中国大陆城市名称。”
- 多城市时：
  - “请先指定一个城市。我当前一次只处理一个中国大陆城市，以保证出处和口径一致。”

## 限制与注意事项

- 不依赖记忆或非官方聚合站做定论，必须使用官方站点实时检索。
- 不回退到省级官网；如果市政府官网和市级主管部门官网都找不到可核验依据，就说明无法确认。
- 不将新闻稿、解读文章、第三方转载当作最终依据；如果它们指向官方文件，可以继续追到原始文件。
- 不做历史沿革、趋势分析、法律解释或政策比较。
- 不为了“看起来像有依据”而引用栏目页、列表页或摘要页；证据必须直达摘录来源。
